# Unmappable Clinical Review — Methodology Archive

**Status**: Complete (2026-03-08)
**Reviewed**: 38,192 / 38,192 items (100%)
**Results**: 22,140 mapped (58%), 16,052 skipped (42%), 0 flagged

## Overview

Item-by-item clinical reasoning review of flowsheet items classified as "unmappable" (category C) by the FCA pipeline. These items had no clinical attributes detected by FCA's regex-based assessment patterns, but ~10% were found to contain clinical value through manual review with OHDSI vocabulary lookups.

## Approach

Each item was reviewed individually with full context (template name, group name, value type, custom list values, units, min/max ranges) and mapped to OMOP standard concepts via vocabulary search tools. No batch regex approaches — every mapping was clinically reasoned.

### Decision categories
- **map**: Item has clinical value, mapped to specific OMOP concept(s) with qualifier concepts (body site, laterality, method) where applicable
- **skip**: Item is non-clinical (admin, template artifact, section header, duplicate)
- **flag**: Ambiguous — resolved in a dedicated flag re-review pass (Session 35)

### Mapping predicates
- **exactMatch**: Direct 1:1 correspondence (e.g., Berg Balance Scale → SNOMED 428203000)
- **narrowMatch**: Source is more specific than target (e.g., specific cath pressure → generic intracardiac pressure)
- **broadMatch**: Source is less specific than target (e.g., institutional ADL scale → FIM)
- **closeMatch**: Semantically close but not hierarchically related

## Orchestration Pattern

Each batch processed ~1,000 items using a 3-tier agent architecture:

1. **Orchestrator** reads config, launches planning agent, then 10 review agents
2. **Planning agent** reads the 1,000-line CSV slice, greps prior mappings, returns a structured plan with agent splits and reusable concept tuples
3. **10 review agents** each handle ~100 items with ±50 context lines for template continuity, perform OHDSI vocab lookups, and write Python scripts

Post-processing: `finalize_batch.sh` runs the scripts, accumulates CSVs, QA-checks, pushes to `clinical_review.csv`, validates, and auto-fixes.
